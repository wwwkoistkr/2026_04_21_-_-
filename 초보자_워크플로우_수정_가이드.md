## 🔹 1-4단계. ⚠️ 삭제할 줄 찾기 (총 2군데, 8줄)

### 🗑️ 삭제 대상 ① — **15번째 줄부터 18번째 줄까지** (총 4줄)

**삭제하기 전 화면:**
```
14      workflow_dispatch: {}
15      # 01_collect 가 성공하면 자동 트리거 (옵션)        ← 지울 것
16      workflow_run:                                     ← 지울 것
```

**작업 방법:**
1. 마우스로 **15번째 줄 맨 앞**을 클릭
2. `Shift` 키를 **누른 채로** **18번째 줄 맨 끝**을 클릭 (4줄이 파랗게 선택됨)
3. 키보드 `Delete` 또는 `Backspace` 키 누르기

### 🗑️ 삭제 대상 ② — **`if: >` 로 시작하는 3줄** (삭제 ① 을 한 뒤 원래 번호로 24~27번째 줄이었던 부분)

**삭제하기 전 화면 (삭제 ① 완료 후):**
```
20      runs-on: ubuntu-latest
21      timeout-minutes: 14
22      # workflow_run 으로 호출됐을 때는 앞 단계가 성공했을 때만 진행   ← 지울 것
23      if: >                                                          ← 지울 것
24        github.event_name != 'workflow_run' ||                        ← 지울 것
25        github.event.workflow_run.conclusion == 'success'             ← 지울 것
26      env:
```

> ⚠️ 삭제 ① 을 먼저 했다면 이 줄들의 번호가 **4줄씩 앞당겨졌을 수도** 있습니다. 상관없습니다. **"# workflow_run 으로 호출됐을 때는..." 주석이 있는 줄부터 "github.event.workflow_run.conclusion == 'success'" 까지** 총 4줄을 삭제하세요.

**작업 방법:**
1. `# workflow_run 으로 호출됐을 때는...` 줄 맨 앞을 클릭
2. `Shift` + `github.event.workflow_run.conclusion == 'success'` 줄 맨 끝 클릭
3. `Delete` 또는 `Backspace`

## 🔹 1-5단계. 확인 — 수정이 잘 되었는지 체크

수정 후 화면이 이런 모양이어야 합니다:

```yaml
on:
  schedule:
    # 06:10 KST (= 21:10 UTC) — collect(06:00) 완료 후 10분 여유
    - cron: '10 21 * * *'
  workflow_dispatch: {}

jobs:
  summarize:
    runs-on: ubuntu-latest
    timeout-minutes: 14
    env:
      FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: "true"
    steps:
      - uses: actions/checkout@v4
```

✅ `workflow_run:` 이라는 단어가 **완전히 사라져야** 합니다
✅ `if: >` 로 시작하는 줄도 **완전히 사라져야** 합니다
✅ Ctrl+F (Windows) 또는 Cmd+F (Mac) 로 `workflow_run` 검색 → **"No results" 나와야 정답**

## 🔹 1-6단계. 💾 저장 (Commit)

1. 화면 **오른쪽 위** 초록색 버튼 **`Commit changes...`** 클릭

4. 두 번째 입력칸 (설명) 은 **비워둬도 됨**
5. 라디오 버튼 **`Commit directly to the main branch`** 가 선택되어 있는지 확인 (기본값)
6. 초록색 **`Commit changes`** 버튼 클릭

# ⚠️ 문제해결 (트러블슈팅)

## 🆘 "Commit changes" 버튼이 회색이고 눌러지지 않아요

- 파일을 실제로 수정한 내용이 없다는 뜻입니다
- 다시 잘 보고 **8줄(파일1) / 6줄(파일2)** 이 정말 삭제되었는지 확인

## 🆘 파일 저장 후에도 메일이 3통 와요

**원인 1: 수정이 제대로 안 된 경우**
- 수정된 파일에서 Ctrl+F 로 `workflow_run` 검색 → 결과가 있으면 추가 삭제 필요

**원인 2: 캐시 문제**
- Actions 탭에서 **Summarize** 와 **Send** 워크플로우를 각각 클릭해서
- 오른쪽 위 **`···`** (점 세 개) → **`Disable workflow`** 했다가 다시 **`Enable workflow`**
- 이러면 캐시가 날아가고 새 설정이 완벽히 반영됩니다

## 🆘 실수로 다른 부분을 지웠어요

걱정 마세요! GitHub 은 모든 변경 이력을 저장합니다.
1. 파일 화면 오른쪽 위 **`History`** 버튼 클릭
2. 실수 직전 커밋을 찾아서 클릭
3. 오른쪽 위 **`···`** → **`Revert`** 로 되돌리기
4. 또는 간단히 저에게 물어보세요 — 제가 올바른 파일 전체 내용을 복사해서 드릴 수 있습니다
